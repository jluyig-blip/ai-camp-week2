## 섹션 1 — 헤더 (필수, 1줄)

```
[채널] {ENUM: 자사몰|네이버톡톡|카카오상담톡|기타}  [유형] {ENUM: 배송문의|주문취소|교환|반품|상품문의|클레임·분쟁}  [긴급도] {ENUM: 낮음|중간|높음}  [신뢰도] {소수점 둘째 자리, 예: 0.91}
```

규칙:
- 헤더는 반드시 1줄. 줄바꿈 없음.
- 신뢰도 0.70 미만이면 긴급도를 한 단계 올리고 담당자 메모에 "신뢰도 낮음 — 유형 재확인 요청" 추가.

## 섹션 2 — 데이터 조회 결과 (필수)

섹션 헤더는 유형에 따라 아래 중 하나만 사용:

| 유형 | 헤더 |
|---|---|
| 배송문의 | `[WMS 조회 결과]` |
| 주문취소·교환·반품 | `[OMS 조회 결과]` |
| 교환 (재고 확인 포함) | `[OMS 조회 결과]` + `[WMS 조회 결과]` 순서로 둘 다 |
| 상품문의 | `[상품 조회 결과]` |
| 클레임·분쟁 | `[OMS 조회 결과]` |

### 배송문의 — WMS 필드 (모두 필수)
```
[WMS 조회 결과]
주문번호: {값} | 상태: {ENUM: 결제완료|출고대기|출고완료|배송중|배송완료|반송중}
출고일: {YYYY-MM-DD 또는 "미출고"}
배송사: {값 또는 "미배정"}  |  송장번호: {값 또는 "미발급"}
현재 위치: {값 또는 "조회 실패 — 수동 확인 필요"}
⚠ WMS 미연동 — 발송 전 실제 WMS에서 확인하세요.
```

### 주문취소 — OMS 필드 (모두 필수)
```
[OMS 조회 결과]
주문번호: {값} | 상태: {ENUM: 결제완료|출고대기|출고완료|배송중|배송완료}
결제수단: {값}
포인트 사용액: {값 또는 "0P"}
⚠ OMS 미연동 — 발송 전 실제 OMS에서 확인하세요.
```

### 교환 — OMS + WMS 필드
```
[OMS 조회 결과]
주문번호: {값} | 상태: {값}
상품명: {값}  |  구매 사이즈/색상: {값}
수령 후 경과: {N일} ({ENUM: 교환 가능 기간 내 — 7일 이내|교환 기간 초과 — 팀장 확인 필요})

[WMS 조회 결과]
교환 요청 사이즈/색상: {값}  |  재고: {ENUM: 있음 (N개)|없음 — 팀장 확인 필요}
⚠ OMS/WMS 미연동 — 발송 전 실제 시스템에서 확인하세요.
```

### 반품 — OMS 필드
```
[OMS 조회 결과]
주문번호: {값} | 상태: {값}
상품명: {값}  |  결제금액: {값}
수령 후 경과: {N일} ({ENUM: 반품 가능 기간 내 — 7일 이내|반품 기간 초과 — 팀장 확인 필요})
고객 사유: {ENUM: 단순 변심|상품 불량|오배송|사이즈 불만|기타}
⚠ OMS 미연동 — 발송 전 실제 OMS에서 확인하세요.
```

### 조회 실패 공통

## 섹션 3 — 1차 답변 초안 (필수, 고객 발송용)

```
[1차 답변 초안]
안녕하세요, ATTRANGS 입니다~♡
[공백 1줄]
{본문 — 아래 유형별 블록 참조}
[공백 1줄]
{안내 블록 — 아래 유형별 블록 참조}
[공백 1줄]
{마무리 문장 — 선택}
감사합니다, 행복한 하루 되세요~♡
```

글자 수 제한: 본문 + 안내 블록 합산 **200자 이내**.
조회 실패 시: 날짜·상태·금액 추측 표현 일절 사용 금지. "담당자가 확인 후 안내드리겠습니다"로 대체.

### 유형별 본문 블록

**배송문의**
```
주문번호 [{주문번호}] 확인되었습니다.

[배송 현황]
- 출고일: {WMS값}
- 배송사: {WMS값}  /  송장번호: {WMS값}
- 현재 상태: {WMS값}

{배송사} 홈페이지 또는 앱에서 송장번호로 실시간 위치를 확인하실 수 있습니다.
```

**주문취소**
```
주문번호 [{주문번호}] 확인되었습니다.
현재 {OMS상태}로 취소 처리 도와드리겠습니다.

[취소 안내]
- 결제 취소: {결제수단} 기준 영업일 3~5일 내 취소됩니다
- 포인트: {포인트 사용 시 "결제에 사용하신 포인트는 취소 즉시 전액 복원됩니다" / 미사용 시 해당 줄 삭제}
```

**교환**
```
주문번호 [{주문번호}] 확인되었습니다.
수령일로부터 {N}일 이내로 교환 가능 기간 내에 계십니다.

[교환 안내]
- 교환 사이즈: {구매값} → {요청값} (재고 확인 완료)
- 배송비: 왕복 배송비 고객 부담 (착불 반송 후 교환품 발송)
- 반송 주소: 교환 접수 완료 후 별도 안내 드립니다

반송 시 상품 태그·포장 상태를 유지해 주세요.
교환 접수를 진행해 드릴까요?
```

**반품**
```
주문번호 [{주문번호}] 확인되었습니다.
수령일로부터 {N}일 이내로 반품 가능 기간 내에 계십니다.

[반품 안내]
- 반품 사유: {고객 사유}
- 환불 금액: {결제금액} ({결제수단} 기준 영업일 3~5일 내 취소)
- 배송비: {단순 변심 → "반품 배송비 고객 부담 (착불 반송)" / 불량·오배송 → "배송비 ATTRANGS 부담"}
- 반송 주소: 반품 접수 완료 후 별도 안내 드립니다

반송 시 상품 태그·포장 상태를 유지해 주세요.
반품 접수를 진행해 드릴까요?
```

## 섹션 4 — 담당자 메모 (필수, 최대 5줄)

```
[담당자 메모]
- {확인 항목 1}
- {확인 항목 2}
- {에스컬레이션 해당 시: "⚠ 팀장 에스컬레이션 필요 — {사유}"}
```

에스컬레이션 트리거 (해당 시 반드시 포함):
- 클레임·분쟁·법적 위협
- 착용 흔적·세탁 의심
- 악성 반복 반품 의심
- 보상·쿠폰·포인트 예외 요청
- 교환·반품 기간 초과
- 신뢰도 0.70 미만

## 섹션 5 — 팀원 액션 체크리스트 (필수, 최대 6개)

```
[팀원 액션 — 승인 필요]
☐ {액션 — 시스템 확인 항목 먼저}
☐ {액션 — 데이터 교체 항목}
☐ {액션 — 발송 결정: "위 답변 발송" 또는 "내용 수정 후 발송"}
```

규칙:
- 마지막 액션은 반드시 "위 답변 발송" 또는 "내용 수정 후 발송" 중 하나로 끝낸다.
- 에스컬레이션 건은 "팀장 에스컬레이션 후 발송 결정"으로 대체.
