# 답변 톤 하네스 이식 가이드 — 정기열 부대표님용

> **출처**: 아뜨랑스 AICAMP 답글 매니저 하네스 (윤시영, AI 하네스 6주반)
> **대상**: 상품 Q&A 응답 + CS 문의 응대 하네스
> **목표**: 우리 답글 매니저의 "톤 게이트(C1~C11) + 피드백 루프"를 정기열님 기존 하네스에 그대로 이식
> **소요**: 약 30분 (복붙 + 문구 치환 수준, 새 코드 없음)

## 0. 한 줄 요약

> **답글 매니저가 만든 답변 퀄리티 = "톤 제약 11개를 통과한 답변만 사람에게 올린다" + "사람이 잔소리한 걸 제약으로 박아서 다음부터 자동 차단한다"의 결과.**
> 이 두 가지(톤 게이트 + 피드백 루프)만 옮기면 도메인이 Q&A/CS여도 똑같이 작동한다.

이식할 핵심 3개는 전부 **문서·프롬프트**다. 코드 이식이 없다.

| 옮길 것 | 형태 | 정기열님이 할 일 |
|---|---|---|
| ① tone-reviewer (C1~C11 게이트) | 체크리스트 프롬프트 | 자기 `tone-reviewer.md`에 붙여넣고 §3 가감표대로 수정 |
| ② risk-checker (위험 8종) | 체크리스트 프롬프트 | 자기 `risk-checker.md`에 붙여넣고 §4 가감표대로 수정 |
| ③ 피드백 루프 방법론 | Round1 → 사람검수 → 제약신설 → Round2 | §1 그대로 1회전 돌리기 |

## 1. 핵심 엔진 — 피드백 루프 5단계 (이게 퀄리티의 진짜 비결)

답변 품질은 좋은 프롬프트 한 방에서 안 나온다. **사람이 반려하고, 반려 사유를 제약으로 굳히는 1회전**에서 나온다. 우리 실제 사례:

```
1) Round 1 — 1차 톤 제약(C1~C5)만으로 답변 초안 생성
   → tone-reviewer: C1~C5 모두 통과 (5/5, "봇 기준 합격")

2) 사람 검수 — 반려.
   봇은 통과시켰지만 사람 눈에는 "AI 냄새" 5종이 보였다:
   - 인사 없이 본문 시작 (벽글)
   - 한 덩어리 단락 (줄바꿈 0개)
   - 사무적 종결 ("공유드릴게요"로 뚝 끊김)
   - 클리셰 ("정말 감사해요", "마음이 따뜻해졌습니다")
   - 큰따옴표 인용 패턴 ("○○"라고 해주신 한 줄에…)

3) 제약 신설 — 위 5종을 막는 제약 C6~C11을 새로 박음

4) Round 2 — 같은 답변을 C1~C11 전체로 다시 생성

5) 재검수 — tone-reviewer: 11/11 = 100% 통과 → 사람 채택
```

**정기열님이 할 일**: 위 과정을 자기 도메인(Q&A 1건 또는 CS 문의 1건)으로 **딱 1회전** 돌린다.
- Round 1 답변을 만들고 → **직접 눈으로 보고 거슬리는 점 3~5개를 적는다** → 그걸 C6~C11처럼 제약으로 박는다 → Round 2.
- 이 "사람이 적은 잔소리 목록"이 곧 정기열님만의 톤 자산이 된다.

> 💡 포인트: Round 1에서 봇이 통과시켜도 **사람이 한 번은 반드시 반려**해본다. 첫 통과를 의심하는 게 이 루프의 핵심이다.

## 2. 답변 결과 카드 — 10필드 (그대로 사용)

모든 답변은 자유 글이 아니라 **카드 1장(10필드)**으로 만든다. 빈칸 0개 원칙.

```
1. 채널 (자사몰 Q&A / CS문의 / 네이버 등)
2. 작성일시
3. 별점 또는 문의유형 (Q&A는 별점 대신 "사이즈/배송/재입고/교환" 등 유형)
4. 본문 요약 (40자 이내)
5. 분류 (배송·품질·사용성·CS·기능요청·칭찬·기타)
6. 감성 (긍정 / 부정 / 중립)
7. 위험 신호 (없음 / 있음 — 세부)   ← risk-checker가 검증
8. 우선순위 (상 / 중 / 하)
9. 답변 초안 (한국어, [확인 필요: ~] 마커 허용)   ← tone-reviewer가 검증
10. 사람 확인 필요 여부 (필요 / 불필요 + 이유 한 줄)
```

> Q&A/CS 변형: 3번 "별점"은 Q&A엔 안 맞으니 **"문의 유형"**으로 치환. 나머지 9개 필드는 그대로.

## 3. tone-reviewer — C1~C11 톤 게이트 (원본 + Q&A/CS 가감표)

답변 초안 9번 필드를 아래 11개 항목으로 **하나도 빠짐없이** 체크한다. 목표 통과율 100%(최소 90%).

### 원본 11개 (우리 답글 매니저)

| # | 제약 | 통과 기준 |
|---|---|---|
| C1 | 호칭 통일 | "아띠님" 1회, "고객님/회원님" 0회 |
| C2 | 보일러플레이트 금지 | "노력하겠습니다 / 만족스러운 / 사랑 부탁 / 따끈한 신상 / 또 방문해주세요" 0건 |
| C3 | 부서+기간+행동 명시 | 막연한 "참고" 금지. "이번 주 / 품질팀 / 검수 강화 요청"처럼 구체 |
| C4 | 본문 단서 1개 이상 인용 | 상대가 쓴 구체 단어(색상·상황·이슈)를 최소 1개 반영 |
| C5 | 보상·할인·재발송 약속 금지 | 금액/환불/할인/재발송 0건 (사람 결재 영역) |
| C6 | 인사로 시작 | "○○님 안녕하세요 :)" 등 첫 줄 인사 |
| C7 | 3~4단락 | 한 덩어리 벽글 금지, 단락 분리 |
| C8 | 마무리 인사 | 계절·상황 맞춤 마무리 |
| C9 | 클리셰 금지 | "마음이 따뜻해졌습니다 / 감동받았습니다 / 큰 힘이 됩니다 / 정말 감사드립니다" 0건 |
| C10 | 사무적 종결어 단독 금지 | "공유드릴게요"로 뚝 끊지 말고 인사로 마무리 |
| C11 | 큰따옴표 인용 패턴 금지 | `"○○"라고 / 이라고 / 한 줄에` 패턴 금지 → 자연스럽게 풀어 인용 |

### ⚠️ Q&A/CS용 가감표 (이게 이식의 핵심 — "그대로"만 주면 안 되는 이유)

리뷰 답글은 **감성·톤**이 핵심이지만, Q&A/CS는 **정확한 답이 1순위**다. 톤만 좋고 정보가 틀리면 더 큰 사고다. 그래서:

**▶ 그대로 유지 (수정 불필요)**
- C2, C9, C10, C11 — AI 냄새 제거 계열. 도메인 무관하게 그대로 강력.
- C1 — 호칭만 정기열님 기준으로 ("아띠님" 또는 합의된 호칭).
- C5 — 오히려 Q&A/CS에서 더 중요 (임의 보상 약속은 사고).

**▶ 살짝 완화**
- C8 (계절 마무리) — 환불 진행상황 같은 **사무적 CS 문의**엔 계절 인사가 과할 수 있음 → **"상황에 맞는 마무리"**로 완화. 따뜻한 Q&A엔 그대로 적용.

**▶ 새로 추가 (Q&A/CS 전용 — 정확성 게이트)**
- **C12 직답 우선**: 질문에 **직접 답**부터 한다. 인사·공감만 하고 정작 답이 없는 "동문서답" 금지.
- **C13 추측 답변 금지**: 재고·일정·스펙·정책을 모르면 단정하지 말고 `[확인 필요: ~]` 마커 → "확인 후 안내드리겠습니다"로 처리. (risk-checker R7과 연동)
- **C14 처리 기한·담당 명시**: CS 문의는 "언제까지 / 누가" 처리하는지 명시 (C3의 CS 버전).

> 정리: **C2·C9·C10·C11은 복붙 그대로**, C1·C5·C8은 가벼운 손질, **C12~C14를 새로 추가**. 이게 30분 작업의 본체다.

## 4. risk-checker — 위험 8종 + 자동 발행 금지 (원본 + 가감표)

답변 본문에서 아래 8종을 **over-detect(놓치느니 과검출)** 편향으로 잡는다. 1개라도 hit이면 카드 10번 = **"사람 확인 필요"** 강제.

| # | 위험 신호 |
|---|---|
| R1 | 환불 요청 |
| R2 | 외부 escalation 위협 (타몰 이탈 / 공개 평판 공격 / 보상 강요 / 법적 표현) |
| R3 | 별점 1~2 (Q&A는 → **"강한 불만 어조"**로 치환) |
| R4 | 욕설·비방 |
| R5 | 공개 협박 |
| R6 | 금전 보상·환불 약속이 **답변 초안에** 포함됨 |
| R7 | 사실 주장(인증·재고·일정·스펙) **단정**이 답변 초안에 포함됨 |
| R8 | 개인정보(실명·주문번호·연락처·주소·이메일) 노출 의심 |

**판정 규칙 (그대로 유지)**: R6·R7·R8 중 하나라도 hit이면 카드 10번 값과 무관하게 **자동 반려** → 답변 초안 재작성.

### ⚠️ Q&A/CS 가감
- **R7(사실 단정)이 가장 중요해진다.** Q&A는 "재입고 언제 돼요?" "이 원단 세탁 가능해요?"처럼 사실 답변이 핵심 → 추측 답변이 곧 사고. R7 hit 기준을 **엄격하게**.
- R3 "별점 1~2"는 Q&A에 별점이 없으니 **"강한 불만/항의 어조"**로 치환.
- 나머지 R1·R2·R4·R5·R6·R8은 그대로.

## 5. 회피 패턴 — 절대 쓰지 말 것 (3종)

우리가 실제로 박멸한 3가지. 정기열님도 똑같이 차단 대상.

- **#1 일괄 답글** — 내용 무관 동일 문구 반복. ("고객님 반가워요~ ^-^ … 노력하겠습니다 / 자주 방문해주세요")
- **#2 막연한 다음 조치** — "디자인 개선에 참고하겠습니다" 같은 빈 약속. 구체 단계 없음.
- **#3 고객이 안 한 말 추가 + 길이 늘이기** — 자의적 칭찬·추측 첨가, "고객님" 호칭.

> Q&A/CS 추가 회피 패턴 후보: **#4 동문서답** (질문 안 읽고 정형 답변), **#5 "확인해보겠습니다"만 하고 끝** (담당·기한 없는 핑퐁).

## 6. 절대 규칙 (Human-in-the-loop)

```
1. AI 자동 발행 금지. AI는 카드(답변 초안)까지만. "발행" 액션은 사람만.
2. risk-checker가 위험 1개라도 잡으면 → 카드 10번 = "사람 확인 필요" 강제.
3. tone-reviewer 통과 + risk-checker 통과 + 사람 채택 → 그때 completed.
```

```
[ ] 6. 통과한 카드 1장 = 정기열님 하네스의 첫 golden 예시로 저장
       (이후 golden 2~3건 모이면 reply-writer 품질이 급상승)
```

## 8. 우리 성과 (참고 — 같은 방법으로 나온 숫자)

| 지표 | 값 |
|---|---|
| C1~C11 톤 통과율 | 11/11 = 100% |

## TONE-REVIEWER — C1~C14 톤 게이트

## RISK-CHECKER — 위험 8종

R1 환불 요청 / R2 외부 escalation 위협 / R3 강한 불만·항의 어조
R4 욕설·비방 / R5 공개 협박
R6 금전 보상·환불 약속이 답변 초안에 포함됨
R7 사실 주장 단정이 답변 초안에 포함됨 ← 가장 엄격하게
R8 개인정보 노출 의심

R6·R7·R8 hit → 자동 반려. 위험 1개라도 hit → 사람 확인 필요 강제.
