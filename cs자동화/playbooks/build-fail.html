severity: high deploy deploy-gate-agent SLA: unblock the push within 30 min
Summary — A deployCommands stage (type-check / build / test) failed, so the pre-push gate blocked the push. If a deploy was imminent, fix it now.

Trigger

  • The pre-push hook failed on a deployCommands stage.
  • A local npm run gate failed.
  • monitor-agent saw the hosting provider report a deploy in an ERROR state.

Diagnosis

  1. Open the latest gate log: data/runs/gate-<ts>/<stage>.log.
  2. The first error line usually has file:line — open it, check recent history (git log -p <file>).
  3. Dependency issue? npm ls / check whether package.json changed; confirm runtime deps aren't in devDependencies.
  4. Import path casing — CI is usually case-sensitive even if your dev machine isn't.
  5. New process.env.X read that isn't in .env.example?

Decision tree

type-check error ├─ single type error → fix directly → re-run the gate ├─ many files affected → delegate a task to frontend-agent / backend-agent └─ shared types changed → cross-check across the agents that depend on them build error ├─ import not found → check path / casing / extension ├─ chunk too large → file a code-splitting task to frontend-agent └─ env var missing → sync .env.example + add it in the hosting console

  • If the same failure pattern recurs (3×) → file a task to add a lint/CI rule that catches it.
