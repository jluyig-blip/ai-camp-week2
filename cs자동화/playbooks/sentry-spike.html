Playbook · error-rate-spike

severity: high monitoring monitor-agent → frontend-agent / backend-agent SLA: triage within 15 min
Summary — The error rate (from Sentry or your error tracker) jumped well past baseline. Check causality with the most recent deploy first.

Trigger

  • monitor-agent's poll saw error rate > ~3× the rolling baseline.
  • More than ~5 new unresolved issues within a short window.

Diagnosis

  1. Open Issues for the last hour, sorted by event count.
  2. Read the top 3 stack traces → find the common origin.
  3. Match the spike start time against the last deploy time.
  4. Diff the suspect commit → assess regression likelihood.
  5. Estimate affected users.

Decision tree

spike origin ├─ recent deploy is clearly the cause │ ├─ hotfix possible in <15 min → fix directly │ └─ hotfix hard → roll back now → split the diagnosis into its own task ├─ external dependency (3rd-party API) │ └─ add retry / circuit breaker; watch their status page └─ user-behaviour change (traffic surge, a launch) └─ benign — file a task to retune the threshold

Escalation

  • More than ~100 users affected → user-facing notice + incident channel.
  • Touches a money path or auth → dispatch backend-agent alongside.

Aftermath

  • Write a postmortem (data/runs/postmortem-<ts>.md) if it had user impact.
  • Retune the alert threshold for that error group.
