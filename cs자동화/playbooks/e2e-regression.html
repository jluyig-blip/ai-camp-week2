Playbook · e2e-regression

severity: high qa deploy-gate-agent → frontend-agent / backend-agent SLA: 1h
Summary — A golden-path end-to-end scenario regressed. Treat as user-impacting until proven otherwise.

Trigger

  • A scheduled E2E run, or the gate's E2E stage, failed a golden-path scenario.
  • The same scenario failed twice in a row → an auto-created task points here (severity high).

Golden paths (priority order — adapt to your app)

#scenarioimpact if it breaks
1sign in (all auth methods)everyone locked out — sev1
2core happy path of the productusers blocked — sev1
3checkout / payment (if any)revenue impact — sev1
4account/admin flowsoperational impact

Diagnosis

  1. Open the E2E log/trace: data/runs/task-<id>/<stage>.log (+ the recorded trace).
  2. From the failure screenshot, decide: visual regression vs logic regression vs flake.
  3. Reproduce locally headed.
  4. Correlate the failure time with the last deploy and with error-tracking events.

Decision tree

failure location ├─ broken UI selector → frontend-agent (add stable test ids) ├─ changed API response → backend-agent (server-side regression) ├─ timeout → environment (cold start) → 3 retries; still failing ⇒ real regression └─ DB state → fixture problem (E2E suite self-check)

Escalation

  • Two golden paths failing at once → freeze deploys (no merges to the main branch).
  • Happening to users right now → page the on-call.

Aftermath

  • Add a sharper assertion to the scenario that broke.
  • Consider promoting a near-miss scenario into the golden-path set.
