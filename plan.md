## 📋 유형별 자동화 설계

---

### 1. 배송문의 (45%) — 최우선 과제

**케이스 A: 주문 전 — "지금 주문하면 언제 와요?"**
```
로직:
WMS 조회 → 재고 - 미발송 = 가용재고
가용재고 > 0 → "오늘 주문 시 N일 도착 가능"
가용재고 = 0 → 지연날짜 테이블 조회 → "X월X일 이후 발송 예정"
```

**케이스 B: 주문 후 단일 주문 — "배송 언제 돼요?"**
```
로직:
주문번호 확인 → WMS 조회
할당 받음 → 출고 예정일 + 택배 운송장 안내
할당 못받음 → 지연날짜 테이블 → 입고 예정일 안내
```

**케이스 C: 주문 후 합포 주문**
```
로직:
전체 품목 수 확인
입고 완료 품목 / 미입고 품목 분리
미입고 지연일 계산
지연 3일 이내 → 전체 출고 대기 안내
지연 3일 초과 → 긴급출고 체크 (주문내역 긴급출고 버튼) → 부분출고 처리
```

**사람 개입 필요:**
- 7일 이상 지연 시 담당자 에스컬레이션
- 고객 불만 강도 높을 시 (욕설/강경 표현)

---

### 2. 반품접수 (24%)

```
로직:
주문번호 확인 → 수령 후 7일 이내 여부 확인
정상 → CJ대한통운 자동 수거 접수
환불금액 계산: 결제금액 - 반품비(3,000원) - 쿠폰 조정
카카오 알림톡 자동 발송 (접수완료 + 환불예정금액)
```

**사람 개입 필요:**
- 7일 초과 반품 요청
- 불량/오배송 (배송비 무료 처리 승인)

---

### 3. 교환접수 (10%)

```
로직:
주문번호 + 교환 옵션 확인
WMS 재고 조회 → 교환 희망 옵션 재고 확인
재고 있음 → 자동 교환 접수 + 수거 안내
재고 없음 → "품절 안내 + 환불 또는 다른 옵션 제안"
```

**사람 개입 필요:**
- 부분취소 후 재주문 케이스
- 재고 없을 시 대안 협의

---

### 4. 주문취소 (12%)

**이지듀 사례 참고: 출고 상태 실시간 검증 핵심**
```
로직:
주문번호 확인 → WMS 출고 상태 조회
상품준비중(출고 전) → 즉시 자동 취소 → 알림톡 발송
배송준비중(포장 시작) → 취소 불가 안내 → 담당자 연결
배송중 → 반품으로 안내
```

**핵심: 출고 시작 후 취소 = 매출 손실 방지**
- 전체취소만 허용 / 부분취소는 담당자 처리

---

### 5. 상품/코디문의 (9%)

**이지듀 사례 참고: 도큐먼트 기반 RAG**
```
로직:
상품 도큐먼트 DB 검색
- 상품 상세 스펙 (소재/사이즈/컬러)
- 코디 매칭 상품 코드
- 사이즈 추천 기준표
자연어 질문 → 도큐먼트 근거 기반 답변
출처(아티클) 함께 노출
```

**구축 필요 데이터:**
- 상품별 코디 매칭 테이블
- 사이즈 추천 기준 (키/체중별)
- 자주 묻는 상품 FAQ

## 🔧 필요 API (개발팀 요청사항)

### WMS API 4개
```json
1. GET /api/inventory/available
   - 입력: 상품코드
   - 출력: { 재고수량, 미발송수량, 가용재고, 입고예정일 }

2. GET /api/order/status
   - 입력: 주문번호
   - 출력: { 주문상태, 할당여부, 택배사, 운송장번호, 출고예정일, 품목별상태[] }

3. POST /api/order/return
   - 입력: { 주문번호, 반품사유, 품목코드[] }
   - 출력: { 접수번호, 수거예정일, 환불예정금액 }

4. POST /api/order/cancel
   - 입력: { 주문번호, 취소사유 }
   - 출력: { 취소가능여부, 취소완료여부, 환불예정금액 }
```

### 채널 API 2개
- 카카오 상담톡 API (수신/발신)
- 네이버 톡톡 API (수신/발신)
