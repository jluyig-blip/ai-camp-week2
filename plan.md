## 후보 3개 개발 단계 배치 + AI 평가  (1주차/weekly)
**my_judgment**

될것같음 

**ai_eval_prompt**

시각으로 정리 개발 8단계 (기획 · 설계 · 개발 · 테스트 · 배포 · 운영) 어디에 배치되는지 실현 가능성 · 반복성 · 업무 효과 · AI 위임 가능성 각각 정리좀

**ai_eval_result**

항목	평가
실현 가능성	매우 높음
반복성	매우 높음
업무 효과	매우 높음
AI 위임 가능성	높음
우선순위	1순위

이유: 반복 문의가 많고, WMS 데이터만 정확히 연결되면 AI가 답변 초안을 빠르게 만들 수 있습니다. 처음부터 완전 자동응답보다 상담원 검수형 AI로 시작하는 것이 가장 안전합니다.

**stage_placement**

가장 먼저 개발해야 할 것은 CS 즉답 보조 에이전트입니다.

처음부터 고객에게 AI가 직접 답변하게 만들기보다는,

WMS 조회 → AI 요약 → 답변 초안 생성 → 상담원 확인 후 발송

이 구조로 시작하는 것이 좋습니다.

그다음 단계에서 교환·반품·취소를 자동 분류하고, 마지막으로 발주 추천까지 확장하면 됩니다.
즉, 순서는 아래가 가장 안정적입니다.

CS 답변 자동화 → 교환·반품·취소 판정 보조 → 발주 추천 자동화

| ④ 사실 오류/추측 | 추측 없음, WMS 실값만 사용 | "담당자가 직접 연락드리겠습니다" — 채널 밖 접촉 약속 (정책 확인 필요) |
| ⑤ AI↔사람 경계 | WMS 데이터 검증 주체 명확 (팀원 확인 후 발송) | 경계 위반 없으나 WMS 연동 후 데이터 정확성 검증 주체 불명확 |

## 6. 3주차 제약 조건 후보 3개

**① WMS 조회 결과 필수 포함**
배송 문의 처리 시 `[WMS 조회 결과]` 섹션을 답변 앞에 반드시 출력한다. 조회 실패 시 "WMS 조회 실패 — 수동 확인 필요"를 명시한다.

**② 날짜·상태 추측 금지**
WMS에서 받은 값만 답변에 사용한다. 조회 결과가 없으면 날짜·상태를 절대 추측하지 않는다.

**③ 채널 밖 접촉 약속 금지**
"전화 드리겠습니다" 등 채널 외부 접촉 약속은 자동 제거한다. 채널 내 응대만 약속한다.
