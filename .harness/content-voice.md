# Content & Voice — UX 카피 원칙

자동화 프로덕트의 UI 문구·에러 메시지·툴팁 작성 가이드.

---

## 말투 (Tone)

- **친근하되 전문적**: 사용자는 대부분 담당자(기획자·검토자). 반말 금지, 지나친 존대도 부담.
- **사용자 관점**: "무엇을 보시겠어요?" > "필터 선택". 사용자가 하려는 행동을 직접 묻기.
- **능동태**: "파일이 다운로드됨" ✗ → "xlsx 다운로드 시작" ✓
- **짧은 문장**: 한 줄에 요점 하나.

---

## 용어 한글화 원칙

기술 용어는 기본적으로 **한글 직관 라벨**로 표시하고, 기술 ID는 필요 시만 접기(`<details>`)로.

| 기술 용어 | 한글 라벨 |
|---|---|
| FACT | 출처·사실 확인 |
| SCHEMA | 서식 오류 |
| BORDERLINE | 애매함 — 사람 확인 |
| SENSITIVE | 민감 표현 가능 |
| CONSISTENCY | 앞뒤 불일치 |
| generating | 생성중 |
| validating | 검증중 |
| passed | 통과 |
| flagged | 플래그 |
| regenerating | 재생성 |
| material | 학습자료 |
| quiz | 퀴즈 |
| practice | 실습 |
| figure_rationale | 위인 선정 배경 |
| blueprint | 기획서 |
| run | Run (그대로 — "실행"은 혼동) |
| component | 컴포넌트 |

> 예외: **`Run`**, **`component_id`**, 모델명(`Claude Opus 4.7`)은 영문 유지 가능. 기술 식별자·고유명사.

---

## 섹션 타이틀

- 섹션 번호 붙이는 방식: `① 코스 입력`, `② 결과값 조절`, `③ 고정 프리셋`
- 작업 흐름 순서대로 ①~⑤ 원형 번호
- 긴 제목 피하기 — 한 눈에 들어오는 3~5글자

---

## 버튼 문구

- **기본 동작 중심**: "저장", "삭제", "취소"
- **맥락 포함 필요 시**: "수정 반영 + 승인 → 생성 시작"
- **아이콘 같이**: `⟳ 이 챕터 전체 재생성`, `⚐ 이 부분 보기`
- **금지**: "OK", "Submit", "Click here"

---

## 에러 메시지 3요소

```
[타입] 한줄 요약
어떤 값이 문제였는지 구체적으로
어떻게 복구할 수 있는지
```

예:
```
❌ blueprint.error: AuthenticationError (401)
invalid x-api-key
→ .env의 ANTHROPIC_API_KEY를 console.anthropic.com에서 재발급한 값으로 교체
```

---

## 토스트 문구

- **성공**: "✓ 승인 완료 — 컴포넌트 생성 시작"
- **정보**: "xlsx 다운로드 시작"
- **실패**: "재생성 실패: HTTP 500"
- 짧게, 결과 위주, 이모지 1개 이하

---

## 툴팁 (title 속성)

- 버튼·배지 상세 설명
- 예: "애매함 — AI 검증자가 통과/미달 경계에서 판정한 항목. 사람 눈으로 최종 확인 필요."
- 1~2문장, 약어·기술어 피함

---

## Placeholder

- **실제 예시 문장** 제시
- 예: `예: 영어학원비 월 30만원 내는 30대 직장인`
- 길이 힌트 포함: `(쉼표로 3~5개)`

---

## 숫자·시간 표기

- **숫자**: `10챕터`, `100문항` (공백 없이 단위 붙임)
- **시간**: `약 15~40초`, `2시간 30분`, `하루 10분`
- **점수**: `점수 92`, `95/100` 대신 `점수 95`

---

## 한자·영문 혼용 규칙

- 한자: 최소 사용 (한글 우선)
- 영문 약어: 대문자 유지 (예: AI, GBB, KPI)
- 단위 영문: 소문자 (예: min, px, MB)

---

## Anti-패턴 (하지 말 것)

- ❌ "에러가 발생했습니다" (어떤 에러? 복구 방법?)
- ❌ "material_1_1_abc123" (사용자에게 의미 없음)
- ❌ "Submit" / "Click me" 영문 일색
- ❌ 한 버튼에 3줄 설명
- ❌ 이모지 남발 (1버튼 1개 이하)
- ❌ 높임말 불일치 ("저장합니다" vs "저장" 혼용)
