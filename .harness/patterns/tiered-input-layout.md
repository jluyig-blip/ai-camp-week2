# 패턴 — Tier 입력 레이아웃 (필수 / 품질 / 운영)

> 자동화 프로덕트 입력 필드가 점점 늘어나는 걸 방지하기 위한 3층 분리 원칙.

---

## 왜 Tier로 나누나

- 신규 사용자가 처음 보는 화면에 20개 필드 → 이탈
- 동시에, 숙련 사용자에겐 세밀한 조정 옵션 필요
- 해결: **기본값으로 충분**하게 + **필요한 사람만 깊이** 들어가게

---

## 3개 Tier

### Tier 1 — 필수 입력 (매 run 새로 입력)
- **개수 기준**: 5개 이하
- **타입**: 주로 텍스트 필드 (자유 입력) + 1~2개 드롭다운
- **예시** (이 프로젝트):
  - 위인 (figure)
  - 주제 (topic)
  - 학습 대상 (target_learner)
  - 학습 목표 (learning_goals)
  - 카테고리 (category)
  - 과정 구조 (parts × chapters_per_part)
  - 예상 학습 시간 (chapter_duration_min)

### Tier 2 — 품질 제어 (기본값 제공, 조정 가능)
- **개수 기준**: 6~10개 이하
- **타입**: 슬라이더, 라디오, 드롭다운 (기본값 세팅된 상태)
- **사용자 행동**: 대부분 안 건드림, 필요 시 1~2개 조정
- **예시**:
  - 위인 활용 강도 (낮음/중간/높음) — 슬라이더
  - 힌트 상세도 (간략/표준/상세) — 슬라이더
  - 검증 엄격도 (fast/standard/strict) — 슬라이더
  - 난이도 프리셋 (입문/일반/고급) — 라디오
  - 퀴즈 수·실습 수 — 숫자 슬라이더
  - 타깃 AI 도구 — 드롭다운

### Tier 3 — 운영 고급 (기본 숨김, 접근 가능)
- **개수 기준**: 제한 없음 (필요시 확장)
- **타입**: 고급 설정 영역 (접혀있는 섹션)
- **사용자 행동**: 처음엔 무시, 운영 중 이슈 생기면 조정
- **예시**:
  - 재시도 상한 (1/3/5)
  - 필수/금지 키워드
  - 참조 자료 업로드
  - 언어 선택
  - 출시 채널 (B2C/B2B)
  - 특정 API 엔드포인트 (사내망)

### 프리셋 (관리자만, 읽기 전용 표시)
- **개수 기준**: 5개 이하 (중요한 것만 노출)
- **표시 형식**: `[라벨]: [값]` 한 줄 요약
- **예시**:
  - 서식 버전: v1.0 (aiworld-main)
  - 톤앤매너: 위인 멘토링 시뮬
  - courseId 규칙: AW-YYYYMMDD-{seq}
  - 루브릭: rubric-v1.yaml

## 기본값 설계 원칙

- **기본값은 실제 잘 나오는 값**으로. "가장 흔한 케이스"에 맞춰 세팅
- 기본값으로 Run 돌려도 80점 이상 결과가 나와야 함
- 그래야 Tier 2·3을 건드릴 이유가 "더 잘하고 싶을 때" 로 국한

---

## 파생 값 실시간 표시

- 입력에서 파생되는 값은 **자동 계산 + 옆에 즉시 표시**
- 예: 파트 5 × 챕터 2 = **10챕터**
- 예: 챕터당 15분 × 10챕터 = **총 2시간 30분**
- 사용자가 입력 중 수치 감각 잡는 데 도움

---

## Anti-패턴

- ❌ 모든 필드를 한 섹션에 몰아넣기
- ❌ 기본값 없이 전부 필수로 받기
- ❌ Tier 3 설정을 Tier 1과 섞어 노출
- ❌ 파생 값(총 시간 등)을 사용자가 직접 계산하게
- ❌ 관리자 프리셋을 일반 사용자도 수정 가능하게
