# 패턴 — 3-Screen 플로우 (Control · Execution · Review)

> 자동화 프로덕트 대부분은 "설정 → 실행 → 검토"의 3단계. 이걸 3개 화면(탭)으로 매핑하고 스테퍼로 연결.

---

## 왜 3-Screen인가

- 각 단계에서 사용자의 인지 부담이 다름
  - Control: **무엇을 만들지 결정**하는 집중
  - Execution: **기다리면서 진행을 관찰**하는 수동적 모드
  - Review: **결과를 평가하고 수정**하는 판단 모드
- 한 화면에 다 욱여넣으면 세 역할 다 부실해짐
- 스테퍼로 현재 위치를 상단에 상시 노출 → "내가 어디 있는지" 명확

---

## Screen 1 · Control (입력 단계)

### 구성 (3개 카드)
1. **① 코스/작업 입력** (매 run 입력, Tier 1 필수)
   - 문자열 입력 위주 (텍스트 필드 4~6개)
   - 드롭다운 (카테고리 등 제한된 선택지)
   - 숫자 입력 (수량, 시간 등)
2. **② 결과값 조절** (품질 제어, Tier 2)
   - 슬라이더 (강도 1~3단계)
   - 라디오 (프리셋 선택)
   - 드롭다운 (도구 선택)
3. **③ 고정 프리셋** (관리자 영역, Tier 3, 읽기전용)
   - 서식 버전, 톤 프리셋, 규칙 파일 경로 — 작은 박스로 축소 표시

### 하단
- Primary 버튼 `생성 시작` + 예상 소요 시간 안내
- 필수 필드 미입력 시 토스트 에러

---

## Screen 2 · Execution (실행 관찰)

### 스테퍼
- 1 입력 ✓ → 2 기획 → 3 생성·검증 → 4 검토 → 5 배포
- 현재 단계 하이라이트

### 섹션 배치 (위에서 아래)
1. **StatsBar** — 5-card 진척 요약 (각 컴포넌트별 ✓N ⚠N …N)
2. **Director 카드** — Blueprint Idle / Loading / Editable Form 3-state
   - Loading 상태: 스피너 + 안내 + skeleton shimmer
   - Form 상태: 사용자 편집 가능 + "승인 + 생성 시작" 버튼
3. **Kanban 뷰** — 컴포넌트 상태별 4컬럼 (생성/검증/통과/플래그)
4. **Event Log** — `<details>` 접이식, 기본 접힘

### 중단 게이트
- 승인 없이 다음 단계 진행 금지
- 승인 시 상태가 `awaiting_approval` → `generating` 으로 전환

## Screen 3 · Review (검토·승인)

### 3-Pane Layout
```
┌──────────┬──────────────┬──────────────┐
│ Nav Tree │ Content Prev │ Flag Panel   │
│  (256px) │   (flex-1)   │   (384px)    │
│          │              │              │
│ 커리큘럼 │ 선택한 컴포  │ 해당 플래그  │
│ 트리 +   │ 넌트의 실제  │ 카드 스택 +  │
│ 플래그   │ 콘텐츠 미리  │ 필터 + 벌크  │
│ 카운트   │ 보기 + 앵커  │ 무시 버튼    │
└──────────┴──────────────┴──────────────┘
```

- 두 divider 드래그로 폭 조절 (localStorage로 기억)
- 좌측 트리에서 컴포넌트 선택 → 중앙·우측 동시 갱신
- 플래그 클릭 → 중앙 콘텐츠의 해당 영역으로 앵커

### 키보드 단축키 (권장)
- `↑↓` 또는 `j k` — Nav tree 이동
- `R` — 선택된 플래그 재생성
- `D` — 선택된 플래그 무시

---

## 화면 간 전환

### 자동 전환
- Control → Generate 버튼 → **Execution 탭 자동 이동** + 로딩 표시
- `run.completed` 이벤트 → **Review 탭 자동 이동** + 토스트 알림
- 기존 run 재접근 시 status 기반 적절한 탭으로

### 수동 탐색
- 사이드바 nav item 3개 (Control / Execution / Review)
- 사이드바 history 리스트 → 특정 run 선택 → 상태에 맞는 탭으로 이동

---

## 자동화 도메인 맞춤 예시

- **콘텐츠 자동 생성** (이 프로젝트): Control = 코스 입력 / Execution = 기획서+컴포넌트 생성 / Review = 플래그 기반 검토
- **법률 문서 검토**: Control = 문서 업로드 + 검토 기준 / Execution = 조항별 AI 분석 / Review = 리스크 플래그 검토
- **마케팅 카피 생성**: Control = 브랜드 톤 + 타깃 / Execution = 30개 변형 생성 / Review = 브랜드 위반 플래그
- **데이터 분류**: Control = 분류 룰 + 샘플 / Execution = 배치 처리 / Review = low-confidence 플래그

각 프로젝트에서 Screen 구성은 동일, 콘텐츠 템플릿만 교체.
