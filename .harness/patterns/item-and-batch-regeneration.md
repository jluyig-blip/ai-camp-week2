# 패턴 — 항목 단위 + 챕터 단위 재생성

> AI가 만든 결과물에 문제가 있을 때, **전체를 다시 돌리지 않고 필요한 부분만 재생성**하는 UX와 백엔드 설계.

---

## 재생성 3가지 Scope

| Scope | 대상 | 트리거 위치 | 컨텍스트 상속 |
|---|---|---|---|
| **item** | 컴포넌트 내부의 1개 항목 (퀴즈 1문제, 실습 1문제) | 플래그 카드의 [재생성] 버튼 | 같은 컴포넌트의 다른 항목 요약 |
| **chapter** | 챕터 1개의 학습자료+퀴즈+실습 전체 | 프리뷰 헤더 "이 챕터 전체 재생성" | 기획서 + 이전 버전 피드백 |
| **component** | 특정 타입 전체 (예: 퀴즈 10챕터 모두) | (관리자 전용) | 기획서 + 루브릭 |

---

## 시작·끝 지점 정의 (이게 핵심)

재생성 루프가 언제 시작하고 언제 끝나는지 **명시적으로** 정의 안 하면 무한히 도는 문제가 생긴다.

### 시작 (Trigger)
- 자동: 1차 검증 실패 → 자동 재생성 (최대 N회, config)
- 수동: 사용자가 플래그 카드 또는 챕터 패널에서 명시적 클릭

### 끝 (Termination)
1. **성공 종료**: 재검증 통과 → 새 버전 저장 + 플래그 resolved
2. **루프 상한**: N회 재시도 후에도 실패 → 상태를 `BORDERLINE`으로 격상 + 사람에게 넘김
3. **사용자 종료**: "무시" 버튼 → `dismissed`로 resolved

### 루프 상한 (N)
- 기본 3회 권장
- 너무 높으면 비용↑·의미 없는 반복
- 너무 낮으면 품질 미달 기회 줄어듦

## 입력 (사용자 피드백)

재생성 시 **자유 텍스트 수정 지시**를 받아 시스템 프롬프트에 주입:

```
[사용자 피드백 — 이번 재생성 시 반드시 반영]
고속터미널만 쓰지 말고 30대 직장인이 평일 저녁 한식 모임 찾는 구체 맥락 추가
```

- 기본값: 빈 문자열 (피드백 없이 단순 재시도)
- 없어도 작동. 있으면 프롬프트에 `_user_feedback` 키로 들어가고 generator가 이를 최우선으로 반영

---

## 컨텍스트 상속 (중요)

재생성 에이전트가 고립된 채로 돌면 엉뚱한 결과. 반드시 다음을 함께 전달:

1. **원 생성자와 동일 모델** (교차검증 매트릭스 유지)
2. **기획서 블루프린트** (챕터 의도 유지)
3. **직전 버전 전체 내용** ("뭘 바꿔야 하는지" 알게)
4. **같은 컴포넌트의 형제 항목들** (consistency 유지 — "이 챕터 다른 문제들은 이렇다")
5. **사용자 수정 지시** (있으면)
6. **플래그 사유 + 가이드** (어느 부분이 어떻게 문제였는지)

---

## 버전 관리 (DB)

- 각 재생성은 새 `component_id`로 삽입 (기존 row 수정 아님)
- `parent_version` FK로 족보 추적
- `regenerations` 테이블에 source / target / scope / user_instruction / 모델 저장
- UI는 기본 **최신 버전만** 표시 (navigator tree). 과거 버전은 "diff / rollback" 페이지로 별도.

## UI 구성

### 항목 단위 (플래그 카드)
```
┌────────────────────────────────┐
│ [애매함] 심각도 중   ⚐ 이 부분 │
│ 학습자료 · 위인 화법 일관성   │
│ 멘토 톤 일관 (교과서 표현 금지)│
│ 전반적으로 설명체가 … 약함.   │
│                                │
│ [수정 지시 입력___________]    │
│ [재생성] [무시]                │
└────────────────────────────────┘
```

### 챕터 단위 (amber 패널)
```
┌─ 챕터 1-1 전체 재생성 — 피드백 반영 ──────┐
│ [피드백 textarea]                         │
│ ☑ 학습자료  ☑ 퀴즈  ☑ 실습               │
│ [재생성 시작]  [취소]                     │
└────────────────────────────────────────────┘
```

- 챕터 단위는 체크박스로 부분 재생성 가능 (예: 학습자료만)
- 순서: material → quiz → practice (뒷 단계가 앞 단계를 docText로 참조)

---

## Anti-패턴

- ❌ 재생성 버튼 누르면 전체 파이프라인 다시 돌기
- ❌ 재시도 횟수 제한 없음 (비용 폭발 위험)
- ❌ 재생성 결과가 기존 버전을 덮어써서 비교 불가
- ❌ 사용자 피드백을 받아놓고 프롬프트에 주입 안 함
- ❌ 재생성 완료 알림 없음 (사용자가 끝난 걸 모름)
