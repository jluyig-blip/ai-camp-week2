# 패턴 — 에이전트 상태 Kanban

실시간 에이전트 생성·검증 상태를 **4-column Kanban 뷰**로 표시하는 패턴.

---

## 컬럼 4개 (고정)

| 컬럼 | 포함 상태 | 색 토큰 |
|---|---|---|
| 생성중 | `generating` / `regenerating` (초기) | `status.generating` (blue) |
| 검증중 | `validating` | `status.validating` (amber) |
| 통과 | `passed` | `status.passed` (emerald) |
| 플래그 | `flagged` / `error` | `status.flagged` (rose) |

> 자동화 프로덕트 대부분이 이 4-state로 충분. 더 늘리지 말 것.

---

## 카드 레이아웃

```
┌─────────────────────────────┐
│ 학습자료 · 1-1     [생성중] │  ← 한글 타입 · 위치 + 상태 배지
│ 'Water'의 기적 — AI에게 내…│  ← 부제 (챕터명)
│ claude → openai             │  ← 생성자 → 검증자 (mono, 회색)
└─────────────────────────────┘
```

### 필수 필드
- **타입** (한글) + **위치** (챕터 번호)
- **상태 배지** (우측 상단)
- **부제** (챕터명 또는 항목명)
- **모델 쌍** (작게, 회색)

### 선택 필드
- 점수 (통과 시)
- 플래그 수 (플래그 시)

---

## 컬럼 헤더

```
● 생성중  ← 색 점 + 라벨
    12   ← 카운트 (mono, 회색)
```

- 색 점은 해당 상태 토큰 색
- 카운트 숫자는 실시간 업데이트

---

## 실시간 업데이트

- WebSocket 이벤트 수신 시 카드 이동
- 전환 애니메이션: fade-out 150ms → 새 컬럼에 fade-in 250ms
- 또는 서버 summary API 주기 polling (3~5초) — WebSocket 안 쓰는 경우

---

## Anti-패턴

- ❌ 5컬럼 이상 (인지 과부하)
- ❌ 같은 카드가 여러 컬럼에 중복
- ❌ 모델 정보 숨김 (교차검증 가시화 실패)
- ❌ 해시 ID (`material-1-1-abc123`)를 그대로 노출
