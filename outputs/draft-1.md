# outputs/draft-1.md — 첫 초안 (WMS 없는 상태)

> 골든 데이터(`golden/input-example.md`)의 문의 #001을 처리한 첫 번째 에이전트 출력.
> WMS 연동 없음 → 배송 상태 실조회 불가 → 한계가 드러나는 것이 이 단계의 목적.

---

## 입력

```
채널: 스마트스토어
접수 시각: 2026-05-17 10:23
주문번호: 12345
고객 문의: 안녕하세요 주문한지 5일됐는데 아직 배송이 안왔어요. 주문번호는 12345입니다.
```

---

## AI 처리 결과

```
[채널] 스마트스토어
[유형] 배송 지연
[긴급도] 높음
[신뢰도] 0.95

[1차 답변 초안]
안녕하세요, 고객님. 문의 주셔서 감사합니다.

주문번호 12345의 배송이 지연되어 불편을 드려 진심으로 사과드립니다.
현재 배송 상태를 확인하고 있으며, 빠른 시일 내로 정확한 배송 일정을 안내해 드리겠습니다.
조금만 기다려 주시면 담당자가 직접 연락드리겠습니다. 감사합니다.

[팀원 액션 — 승인 필요]
□ 위 답변 발송
□ 내용 수정 후 발송

[담당자 메모]
- WMS 미연동: 실제 배송 상태 조회 불가 → 수동으로 택배사 조회 필요
- 주문번호 12345로 송장 확인 후 예상 도착일 답변에 직접 기입
- 5일 이상 미배송 → 지연 보상 정책 적용 여부 확인 필요
- 분실 가능성 있을 경우 팀장 보고
```

## 이 초안의 한계 (골든 출력과의 차이)

| 항목 | 골든 출력 | 이번 초안 |
|---|---|---|
| 배송 상태 | WMS 조회 → 간선상차 (2026-05-16 23:00) | "확인 중" — 실제 데이터 없음 |
| 예상 도착일 | 내일(5/18) 오전 중 | 기재 불가 |
| 답변 구체성 | 실제 상태 + 날짜 안내 | 막연한 "빠른 시일 내" |
| 담당자 부담 | 발송만 승인하면 됨 | 배송 조회 + 기입까지 해야 함 |
