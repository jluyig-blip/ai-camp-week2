# 초안 ↔ 골든 격차 분석

> `outputs/draft-1.md` vs `golden/output-example.md` 비교 (문의 #001 — 스마트스토어 배송 지연)

---

## 1. 빠진 정보 (골든엔 있는데 초안엔 없는 것)
- **WMS 조회 결과**: 골든 출력은 배송사·송장번호·현재 위치·예상 도착일을 실제 값으로 제공. 초안은 "확인 중" 한 줄
- **예상 도착일**: 골든은 "내일(5/18) 오전 중"으로 명시. 초안은 날짜 없음
- **현재 배송 단계**: 골든은 "간선상차 완료" 단계 정보 포함. 초안은 없음

## 2. 구조 차이 (순서·섹션·형식이 다른 부분)
- 골든 출력은 `[WMS 조회 결과]` 섹션이 답변 초안 앞에 위치 → 담당자가 데이터 먼저 확인 후 답변 검토 가능
- 초안은 WMS 섹션 없이 바로 답변으로 넘어감 → 담당자가 별도로 조회해야 하는 추가 작업 발생

## 3. 톤 차이 (말투·길이·격식이 다른 부분)
- 골든: "현재 간선 상차 완료 상태로, 내일(5/18) 오전 중 배송 완료 예정입니다" → 구체적·신뢰감
- 초안: "빠른 시일 내로 정확한 배송 일정을 안내해 드리겠습니다" → 막연·불안감 줄 수 있음
- 길이는 비슷하나 구체성 차이가 큼

## 4. 사실 오류 / 추측 (초안이 입력에 없는 걸 만들어낸 부분)
- 초안은 추측성 내용 없음 (무난한 대신 내용이 비어있음)
- "담당자가 직접 연락드리겠습니다" — 스마트스토어 채널에서 전화 연락을 약속하는 건 정책 확인 필요 (채널 밖 접촉 여부)

## 5. AI↔사람 경계 위반 (사람이 봐야 할 걸 AI가 멋대로 처리한 부분)
- 이번 초안에서 경계 위반은 없음
- 단, WMS 연동 후에는 AI가 조회한 배송 데이터를 그대로 답변에 넣기 때문에 **데이터 정확성 검증 주체**를 명확히 해야 함
- "WMS 데이터 = 사실" 로 AI가 단정하면 안 됨 → 팀원이 발송 전 한 번 확인하는 구조 필요

## 6. 그 외 어색한 점
- 5일 미배송이면 **지연 사과 + 구체적 다음 단계**가 필요한데, 초안은 사과만 있고 다음 단계가 없음
- 긴급도 "높음" 분류가 맞는데, 답변 내용은 긴급하지 않은 표준 문구 수준

---

## 7. 3주차 제약 조건 후보

1. **WMS 조회 결과 필수 포함 규칙**: 배송 문의 처리 시 반드시 `[WMS 조회 결과]` 섹션을 답변 앞에 출력. 조회 실패 시 "WMS 조회 실패 — 수동 확인 필요" 명시
2. **날짜·상태 추측 금지**: WMS에서 받은 값만 답변에 사용. 조회 결과 없으면 날짜·상태 절대 추측하지 않음
3. **긴급도 높음 → 다음 단계 명시 규칙**: 긴급도 높음 케이스는 답변에 반드시 "다음 단계(언제까지 어떻게)"를 1줄 이상 포함
4. **채널 밖 접촉 약속 금지**: "전화 드리겠습니다" 등 채널 외부 접촉 약속은 자동 제거. 채널 내 응대만 약속

## 한 줄 회고
첫 초안에서 가장 크게 빗나간 한 가지는 **WMS 데이터 없이 막연한 안내 문구만 생성한 것** 이었고,
그 이유는 **컨텍스트(실시간 배송 데이터)가 부족** 했기 때문이다.
→ 3주차 핵심 과제: WMS 어댑터 연동 + 조회 결과 포함 규칙 추가
