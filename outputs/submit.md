## 1. 이번 주 산출물 한 줄

네이버 톡톡·카카오 상담톡 고객 문의를 AI가 유형 분류·데이터 조회 후 자동 답변하는 CS 에이전트 — AI는 조회·분류·Tier1 자동 발송을 맡고, 사람은 반품·교환 승인 및 분쟁·예외 케이스를 직접 처리한다.

---

## 2. 실제 입력 예시

- 채널: 스마트스토어
- 접수: 2026-05-17 10:23
- 주문번호: 12345

> "안녕하세요 주문한지 5일됐는데 아직 배송이 안왔어요. 주문번호는 12345입니다."

---

## 3. 이상적인 출력 예시

```
[채널] 스마트스토어  [유형] 배송 지연  [긴급도] 높음

[WMS 조회 결과]
배송사: CJ대한통운 | 송장: 123-4567-8901
현재 위치: 수도권 간선터미널 (간선상차 완료, 2026-05-16 23:00)
예상 도착일: 내일(5/18) 오전 중

[1차 답변 초안]
안녕하세요, 고객님. 배송이 지연되어 불편을 드려 죄송합니다.
현재 간선 상차 완료 상태로, 내일(5/18) 오전 중 배송 완료 예정입니다.

[담당자 메모]
- 5일 이상 미배송 → 지연 보상 정책 적용 여부 확인
- 분실 가능성 있을 경우 팀장 보고 후 재발송/환불 안내
```

## 4. AI 첫 초안

```
[채널] 스마트스토어  [유형] 배송 지연  [긴급도] 높음  [신뢰도] 0.95

[1차 답변 초안]
안녕하세요, 고객님. 주문번호 12345의 배송이 지연되어 불편을 드려 진심으로 사과드립니다.
현재 배송 상태를 확인하고 있으며, 빠른 시일 내로 정확한 배송 일정을 안내해 드리겠습니다.
조금만 기다려 주시면 담당자가 직접 연락드리겠습니다.

[팀원 액션 — 승인 필요]
□ 위 답변 발송
□ 내용 수정 후 발송

[담당자 메모]
- WMS 미연동: 실제 배송 상태 조회 불가 → 수동으로 택배사 조회 필요
- 주문번호 12345로 송장 확인 후 예상 도착일 직접 기입
```

## 5. 초안↔골든 격차 분석

| 기준 | 골든 출력 | AI 첫 초안 |
|---|---|---|
| ① 빠진 정보 | WMS 조회 결과(배송사·송장·위치·도착일) 포함 | "확인 중" 한 줄 — 실데이터 없음 |
| ② 구조 차이 | `[WMS 조회 결과]` 섹션이 답변 앞에 위치 → 담당자가 데이터 먼저 확인 | WMS 섹션 없이 바로 답변 → 담당자가 별도 조회 필요 |
| ③ 톤·구체성 | "내일(5/18) 오전 중 배송 완료 예정" — 날짜 명시, 신뢰감 | "빠른 시일 내" — 막연, 불안감 유발 가능 |
| ④ 사실 오류/추측 | 추측 없음, WMS 실값만 사용 | "담당자가 직접 연락드리겠습니다" — 채널 밖 접촉 약속 (정책 확인 필요) |
| ⑤ AI↔사람 경계 | WMS 데이터 검증 주체 명확 (팀원 확인 후 발송) | 경계 위반 없으나 WMS 연동 후 데이터 정확성 검증 주체 불명확 |

## 6. 3주차 제약 조건 후보 3개

**① WMS 조회 결과 필수 포함**
배송 문의 처리 시 `[WMS 조회 결과]` 섹션을 답변 앞에 반드시 출력한다. 조회 실패 시 "WMS 조회 실패 — 수동 확인 필요"를 명시한다.

**② 날짜·상태 추측 금지**
WMS에서 받은 값만 답변에 사용한다. 조회 결과가 없으면 날짜·상태를 절대 추측하지 않는다.

**③ 채널 밖 접촉 약속 금지**
"전화 드리겠습니다" 등 채널 외부 접촉 약속은 자동 제거한다. 채널 내 응대만 약속한다.
