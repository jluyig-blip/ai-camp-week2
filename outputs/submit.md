## 1. 이번 주 산출물 한 줄

네이버 톡톡·카카오 상담톡 고객 문의를 AI가 유형 분류·데이터 조회 후 자동 답변하는 CS 에이전트 — AI는 조회·분류·Tier1 자동 발송을 맡고, 사람은 반품·교환 승인 및 분쟁·예외 케이스를 직접 처리한다.

## 2. 실제 입력 예시

- 채널: 자사몰
- 접수: 2026-05-17 11:05
- 주문번호: 718****27

> "방금 주문했는데 사이즈를 잘못 선택한 것 같아요. 아직 결제만 됐고 출고는 안 됐을 텐데 취소 가능한가요? 포인트도 같이 결제했는데 포인트는 다시 돌아오나요?"

---

## 3. 이상적인 출력 예시

```
[채널] 자사몰  [유형] 주문취소  [긴급도] 중간

[OMS 조회 결과]
주문번호: 718****27 | 상태: 결제완료 (출고 전)
결제수단: 카드 + 포인트 혼합결제
포인트 사용액: 2,000P

[1차 답변 초안]
안녕하세요, ATTRANGS 입니다~♡

주문번호 [718****27] 확인되었습니다.
현재 출고 전 상태로 취소 처리 도와드리겠습니다.

[취소 안내]
- 결제 취소: 카드 결제분은 카드사 기준 영업일 3~5일 내 취소됩니다
- 포인트: 결제에 사용하신 포인트는 취소 즉시 전액 복원됩니다

다른 사이즈로 재주문을 원하시면 포인트 복원 확인 후 재주문해주세요.
감사합니다, 행복한 하루 되세요~♡

[담당자 메모]
- OMS 취소 처리 완료
- 포인트 복원 완료 확인
```

## 4. AI 첫 초안

```
[채널] 자사몰  [유형] 주문취소  [긴급도] 중간  [신뢰도] 0.91

[1차 답변 초안]
안녕하세요, 고객님. 주문 취소 요청 확인했습니다.
현재 출고 전 상태이므로 취소 처리가 가능합니다.
취소 완료 후 결제하신 금액은 카드사 정책에 따라 환불됩니다.

[팀원 액션 — 승인 필요]
□ 위 답변 발송
□ 내용 수정 후 발송

[담당자 메모]
- OMS 미연동: 포인트 사용 여부 및 복원 가능 여부 수동 확인 필요
- 포인트 복원 안내 누락 → 수동으로 포인트 복원액 기입 후 발송
```

## 5. 초안↔골든 격차 분석

| 기준 | 골든 출력 | AI 첫 초안 |
|---|---|---|
| ① 빠진 정보 | WMS 조회 결과(배송사·송장·위치·도착일) 포함 | "확인 중" 한 줄 — 실데이터 없음 |
| ② 구조 차이 | `[WMS 조회 결과]` 섹션이 답변 앞에 위치 → 담당자가 데이터 먼저 확인 | WMS 섹션 없이 바로 답변 → 담당자가 별도 조회 필요 |
| ③ 톤·구체성 | "내일(5/18) 오전 중 배송 완료 예정" — 날짜 명시, 신뢰감 | "빠른 시일 내" — 막연, 불안감 유발 가능 |
| ④ 사실 오류/추측 | 추측 없음, WMS 실값만 사용 | "담당자가 직접 연락드리겠습니다" — 채널 밖 접촉 약속 (정책 확인 필요) |
| ⑤ AI↔사람 경계 | WMS 데이터 검증 주체 명확 (팀원 확인 후 발송) | 경계 위반 없으나 WMS 연동 후 데이터 정확성 검증 주체 불명확 |

## 6. 3주차 제약 조건 후보 3개

**① WMS 조회 결과 필수 포함**
배송 문의 처리 시 `[WMS 조회 결과]` 섹션을 답변 앞에 반드시 출력한다. 조회 실패 시 "WMS 조회 실패 — 수동 확인 필요"를 명시한다.

**② 날짜·상태 추측 금지**
WMS에서 받은 값만 답변에 사용한다. 조회 결과가 없으면 날짜·상태를 절대 추측하지 않는다.

**③ 채널 밖 접촉 약속 금지**
"전화 드리겠습니다" 등 채널 외부 접촉 약속은 자동 제거한다. 채널 내 응대만 약속한다.
