# 고객문의-자동응답 — Service Spec

> `.harness/checklist.md` 의 8개 차원에 대한 답을 모은다.
> Claude Code/Codex 가 plan.md(1주차 계획)에서 끌어낼 수 있는 만큼 채우고, 모르는 건 한 번에 물어본다.

## A. 목적·전략 (Why)
- ① 타겟 유저: 인터넷 쇼핑몰(ATTRANGS) 고객지원팀 상담원 및 팀장
- ② 푸는 통증: WMS/OMS를 매번 수동 조회하며 답변을 작성하느라 응대 시간이 길어지는 반복 업무. 배송·취소·교환·반품 등 유형이 많고, 포인트·혼합결제 등 변수가 있어 실수 위험도 높음.
- ③ 성공 지표: 1차 답변 초안 생성 시간 단축 / 상담원 1인당 처리 건수 증가 / 초안 수정 없이 그대로 발송 가능한 비율(Tier1 자동화율)
- ④ MVP 범위 (안 할 것 명시): 직접 발송 없음 · DB/로그인/인증 없음 · 라우팅 없음 · 관리자 페이지 없음 (5~6주차). 현재는 입력 → 초안 생성 → 상담원 검토 한 페이지만.

## B. 정보구조·콘텐츠 (What)
- ① 사이트맵 (이번 2주차엔 1페이지): `/` — 고객 문의 입력 → AI 답변 초안 출력 단일 화면
- ② 콘텐츠 모델 (입력 타입 / 출력 타입):
  - 입력: 채널(자사몰/네이버톡톡/카카오상담톡) · 접수일시 · 주문번호 · 고객 문의 원문
  - 출력: `[채널/유형/긴급도]` 분류 + `[OMS 조회 결과]` 섹션 + `[1차 답변 초안]` + `[담당자 메모]`
- ③ 작성 주체: AI(Claude)가 초안 생성 → 상담원이 검토 후 발송 결정
- ④ 거버넌스: 반품·교환 승인·분쟁·VIP·악성고객 건은 반드시 팀장 또는 상담원이 직접 처리. AI는 Tier1(일반 배송·취소·포인트 문의)만 초안 생성.

## C. 사용자·계정 (Who)
- ① 인증: 없음 (5~6주차에 추가)
- ② 역할·권한: 상담원 — 초안 검토·발송 결정 / 팀장 — 예외 케이스 최종 승인
- ③ 온보딩: `golden/` 데이터로 초안 품질 직접 확인 후 실무 투입
- ④ 데이터 격리: MVP에선 없음. 고객 개인정보(주문번호 등)는 로컬에서만 처리, 외부 전송 없음.

## E. 통합·자동화 (Extend)
- ① 외부 통합: OMS/WMS 데이터 조회(⚠️ MVP에선 모의 데이터 사용, 실 연동은 5주차)
- ② API·에이전트 접근: Claude API — `prompts/system.md` 를 system 파라미터로 전달 (5주차 연결)
- ③ 백그라운드 작업: 없음 (MVP)
- ④ 콘텐츠 파이프라인: 고객 문의 텍스트 → 유형 분류(AI) → OMS/WMS 조회(모의) → 답변 초안(AI) → 상담원 검토 → 발송
- ⑤ 시스템 프롬프트: `prompts/system.md` — 3주차 제약 조건 3개 + 에스컬레이션 기준 + 브랜드 톤 + AI↔사람 경계 명시

## F. 디자인·UX (Look)
- ① 디자인 토큰: `--brand:#0F172A` (다크 네이비) / `--accent:#F43F5E` (로즈 핑크) — ATTRANGS 패션 브랜드 톤
- ② 컴포넌트: 2-column 레이아웃 (입력 패널 + 출력 패널) · textarea · 실행 버튼 · 상태 배지(유형·긴급도·신뢰도)
- ③ 반응형·접근성: `md:grid-cols-2`, 모바일 단일 컬럼. 한글 폰트(Pretendard) 기반.
- ④ 상태 표현: 로딩 → 버튼 비활성 + 스피너 / 빈 입력 → alert / 성공 → 출력 패널 갱신 / 에러 → 에러 메시지 인라인
- ⑤ 카피·톤: 친근하고 정확함. 브랜드 답변은 "ATTRANGS 입니다~♡" 스타일 유지. 상담원용 메모는 간결·사무적.
