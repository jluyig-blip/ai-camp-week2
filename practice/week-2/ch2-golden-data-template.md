# 골든 데이터 템플릿 — 내 과제용

> 골든 데이터 = **"이상적인 입력 한 묶음 + 그 입력에 대한 이상적인 출력 1~2건"**.
> 에이전트가 이걸 보고 "이런 입력엔 이런 출력을 내야 하는구나" 를 배운다.
> 1주차에 가르친 "AI 가 못하는 것(맥락·최신정보)" 을 메우는 게 바로 이 데이터.

## 좋은 골든 데이터의 4가지 조건
1. **실제 형태** — 가공·요약·정리하지 말고, 실제로 들어올 데이터 그대로. 지저분해도 OK (오히려 그게 진짜).
2. **충분한 양** — 입력 예시 최소 1건, 가능하면 3~5건. 출력 예시 1~2건.
3. **출력이 핵심** — "이상적인 출력"이 빠지면 에이전트가 목표를 모른다. 직접 잘 쓴 결과물 1개를 반드시 포함.
4. **다양성** — 쉬운 케이스 + 까다로운 케이스 둘 다 (출력은 1~2건이면 충분).

## 빈 양식

### `golden/input-example.md` — 실제 입력 데이터
```
(여기에 본인 과제가 실제로 받을 입력 데이터를 그대로 붙여넣기 — 1건 이상.
 표·로그·메일·CSV·문서 등 형태 무관, 실제 그대로.)
```

### `golden/output-example.md` — 이상적인 출력
```
(위 입력에 대해 "이렇게 나오면 완벽하다" 싶은 결과물 — 직접 잘 쓴 것 1~2건.
 형식·구조·톤·길이까지 "이게 정답이다" 싶은 모습으로.)
```

### (선택) `golden/input-2.md` — 까다로운 케이스 입력
```
(엣지 케이스·예외 상황이 섞인 입력 1건. 첫 초안 테스트용으로 좋다.)
```

## 직무별 예시 (참고 — 본인 것으로 바꾸세요)
| 과제 유형 | input-example | output-example |
|---|---|---|
| 리뷰 자동 분류·답글 초안 | 실제 리뷰 20건 (별점·본문·날짜 그대로) | 그 중 5건의 분류 + 답글 초안 모범 (환불·법적 이슈 케이스는 "사람 검토 필요" 표시) |
| 영업 사전 브리핑 | 거래처 1곳의 과거 미팅 기록 + 기본 정보 | 그 거래처용 브리핑 글 1편 (다음 방문 시 알아야 할 것) |
| 면접질문 생성 | 직무 1개의 NCS 자료(역량 정의) | 그 직무의 모범 면접질문 5개 + 평가 포인트 |
| 아침 기술 리서치 | AI 아티클 5개 (제목·요약·링크) | "쓸만한 것 2개" 선별 + 왜 쓸만한지 + 적용 아이디어 |
| 양식 자동 변환 | 원본 엑셀 1개 (실제 데이터) | 변환 결과 엑셀(또는 표) 1개 — *시스템 업로드는 제외, 변환 결과까지만* |
| 고용지원금 검토 | 사업장 데이터 1개 | 모범 검토 결과 1건 (해당 지원금·금액·근거) — *실제 신고는 제외, 검토 결과까지만* |
